Question-2: What is the difference between innerHTML, innerText, and textContent ?

innerHTML sets the HTML content (including tags) of an element, while innerText gets or sets only the visible text, accounting for CSS styles. In contrast, textContent sets all text inside an element, including hidden text, without interpreting HTML tags. Use them based on whether you need to handle HTML, visible text, or raw text content.

Question-3: What is event delegation in the DOM?

Event delegation is a technique in the DOM where you attach a single event listener to a parent element instead of individual listeners to multiple child elements. This works because events bubble up from the child to the parent, allowing the parent to handle events for its children dynamically. It improves performance and simplifies managing events for dynamically added or removed elements.

Question-4: What is event bubbling in the DOM?

Event bubbling is a phase in the DOM event flow where an event triggered on a child element propagates upward through its ancestors. For example, clicking a button triggers the event on the button first, then it bubbles up to its parent elements. This allows parent elements to handle events from their children, enabling techniques like event delegation.
